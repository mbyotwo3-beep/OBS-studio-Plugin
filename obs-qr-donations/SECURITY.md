# Security Policy

## Supported Versions

We take security seriously. Here's our policy for addressing security vulnerabilities:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you've found a security vulnerability in the OBS QR Donations plugin, please report it to our security team by emailing [security@example.com](mailto:security@example.com). You should receive a response within 48 hours.

### When to Report a Vulnerability

Please report any vulnerabilities that could harm the users of this software, including:

- Remote code execution
- Authentication bypass
- Data leakage
- Privilege escalation
- Financial impact
- Cryptographic weaknesses

### What to Include in Your Report

To help us better triage your report, please include:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Any proof-of-concept code
4. Impact of the vulnerability
5. Suggested mitigation or fix (if known)

## Our Commitment

- We will acknowledge receipt of your report within 48 hours
- We will work with you to understand and validate the issue
- We will keep you informed of the progress towards resolving the issue
- We will credit you for your discovery (unless you prefer to remain anonymous)

## Security Best Practices

### For Users

- Always use the latest version of the plugin
- Keep your OBS Studio installation up to date
- Only download the plugin from official sources
- Be cautious when entering sensitive information

### For Developers

- Follow secure coding practices
- Keep dependencies up to date
- Use the principle of least privilege
- Validate and sanitize all user inputs
- Use secure communication channels (HTTPS, WSS)

## Responsible Disclosure

We follow responsible disclosure practices. We ask that you:

- Allow us a reasonable amount of time to address the issue before any public disclosure
- Make a good faith effort to avoid privacy violations, data destruction, and service interruptions
- Not exploit a security issue for any reason

## Security Updates

Security updates will be released as patch versions (e.g., 1.0.1, 1.0.2). We recommend always using the latest version of the plugin.
